Descripción del Proyecto: Aplicación CRUD con Django

Este proyecto consiste en una aplicación web desarrollada con el framework Django, que implementa un sistema de gestión de tareas (CRUD: Crear, Leer, Actualizar, Borrar). La aplicación permite a los usuarios autenticados gestionar sus tareas, visualizar las tareas completadas y pendientes, y manejar su perfil de usuario.

Características Principales:

Registro e Inicio de Sesión:

  • Los usuarios pueden registrarse en la aplicación y crear una cuenta.
  • Los usuarios pueden iniciar sesión y cerrar sesión.

Gestión de Tareas:

  • Crear Tareas: Los usuarios pueden crear nuevas tareas con campos como título, descripción, fecha de finalización e importancia.
  • Visualizar Tareas: Los usuarios pueden ver una lista de sus tareas pendientes y completadas.
  • Actualizar Tareas: Los usuarios pueden actualizar detalles de las tareas existentes.
  • Borrar Tareas: Los usuarios pueden borrar tareas.

Autenticación y Autorización:

  • La aplicación asegura que solo los usuarios autenticados puedan acceder a las funcionalidades de gestión de tareas.
  • Las tareas están vinculadas a los usuarios, de modo que cada usuario solo puede gestionar sus propias tareas.

Interfaz de Usuario con Bootstrap:

  • La interfaz de usuario utiliza Bootstrap para un diseño responsivo y estilizado.
  • El menú de navegación se adapta según el estado de autenticación del usuario (mostrar opciones relevantes para usuarios autenticados y no autenticados).

Estructura del Proyecto:

Modelos:

  • Tarea: Modelo que representa una tarea con campos como título, descripción, fecha de creación, fecha de finalización, importancia y usuario propietario.

Formularios:

  • FormularioTareas: Formulario basado en el modelo Tarea para la creación y actualización de tareas.

Vistas:

  • inicio: Vista de inicio de la aplicación.
  • registro: Vista para el registro de nuevos usuarios.
  • iniciarSesion: Vista para el inicio de sesión.
  • cerrar_sesion: Vista para cerrar la sesión.
  • tareas: Vista para visualizar las tareas pendientes del usuario autenticado.
  • creacionTareas: Vista para la creación de nuevas tareas.
  • detalleTareas: Vista para visualizar y actualizar los detalles de una tarea específica.
  • tareasFinalizadas: Vista para visualizar las tareas completadas.

Plantillas:

  • templates.html: Plantilla base que utiliza Bootstrap para la estructura y el diseño.
  • registro.html, iniciar_sesion.html, tareas.html, creacion_tareas.html, tareas_detalles.html: Plantillas específicas para cada funcionalidad, extendiendo la plantilla base.

Tecnologías Utilizadas:

  • Backend: Django
  • Frontend: HTML, CSS, Bootstrap
  • Base de Datos: SQLite (puede ser reemplazada por cualquier base de datos compatible con Django)
  • Autenticación y Autorización: Sistema de autenticación integrado de Django
